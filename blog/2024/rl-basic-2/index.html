Reinforcement Learning 기초 개념 (2): Algorithm Classification

Created on July 06, 2024

2024   ·   Reinforcement Learning   Taxonomy   ·   ML/DL Theory

Reinforcement Learning algorithm은 agent가 무엇을 학습하는지, environment model을 사용하는지, 그리고 data를 어떻게 재사용하는지에 따라 여러 방식으로 분류할 수 있다. 이 글에서는 REINFORCE algorithm을 이해하기 전에 필요한 algorithm classification을 정리하였다.

Figure 1. Reinforcement Learning algorithm의 model-free, model-based, policy optimization, Q-learning 기반 분류.

Concept

Reinforcement Learning algorithm은 agent가 environment와 상호작용하며 얻은 experience를 이용해 더 높은 return을 얻는 policy를 학습하는 방법이다.

기초 개념 (1)에서는 Reinforcement Learning problem을 MDP로 정의하고, return $R(\tau)$와 objective $J(\pi)$를 정리하였다. 이번 글에서는 algorithm이 어떤 function을 중심으로 학습하는지에 따라 Reinforcement Learning algorithm을 분류한다.

Reinforcement Learning에서 agent가 학습할 수 있는 주요 대상은 다음과 같다.

Learning target Meaning Related algorithm family
Policy $\pi$ state에서 action을 선택하는 함수 Policy-based algorithm
Value function $V^{\pi}(s)$, $Q^{\pi}(s,a)$ state 또는 state-action pair의 expected return을 추정하는 함수 Value-based algorithm
Transition function $P(s_{t+1} \mid s_t, a_t)$ environment의 transition dynamics를 예측하는 함수 Model-based algorithm

Policy를 직접 학습하면 agent는 state $s_t$에서 어떤 action $a_t$를 선택할지 바로 결정할 수 있다. Value function을 학습하면 agent는 action의 가치를 비교하여 좋은 action을 선택할 수 있다. Transition function을 학습하면 agent는 실제 environment에서 행동하지 않고도 future trajectory를 예측할 수 있다.

Algorithm Classification

Deep Reinforcement Learning algorithm은 크게 policy-based, value-based, model-based algorithm으로 나눌 수 있다. 여기에 policy와 value function을 함께 사용하는 Actor-Critic 계열, model과 value와 policy를 함께 사용하는 hybrid 계열이 추가된다.

Category Main learning target Example
Policy-based Policy $\pi$ REINFORCE
Value-based Value function $Q^{\pi}(s,a)$ SARSA, DQN, Double DQN
Model-based Transition model $P(s_{t+1} \mid s_t, a_t)$ MPC, MCTS
Value + Policy-based Policy $\pi$ and value function Actor-Critic, A2C, A3C, PPO, SAC
Model + Value + Policy-based Model, value function, policy Dyna-Q, AlphaZero, MuZero


이 분류는 algorithm이 expected return $J(\pi)$를 최대화하기 위해 어떤 function을 직접 학습하는지에 따라 결정된다.

Policy-based Algorithm

Policy-based algorithm은 좋은 policy $\pi$를 직접 학습하는 방법이다. 좋은 policy는 높은 return을 갖는 trajectory $\tau$를 생성할 수 있어야 한다.

Policy-based algorithm의 objective는 다음과 같이 표현할 수 있다.

\[J(\pi) = \mathbb{E}_{\tau \sim \pi} \left[ R(\tau) \right]\]

Policy-based algorithm은 policy parameter $\theta$를 사용하여 policy를 $\pi_{\theta}$로 표현한다.

\[\pi_{\theta}(a \mid s) = P(a_t = a \mid s_t = s; \theta)\]

이때 학습의 목표는 expected return을 최대화하는 parameter $\theta$를 찾는 것이다.

\[\theta^* = \arg\max_{\theta} J(\pi_{\theta})\]

Policy-based algorithm은 action space가 discrete인 경우뿐 아니라 continuous action space에도 적용할 수 있다. 또한 objective $J(\pi_{\theta})$를 직접 optimize한다는 점에서 직관적이다.

대표적인 policy-based algorithm은 REINFORCE이다. REINFORCE는 policy gradient를 이용해 policy parameter $\theta$를 update하는 가장 기본적인 Monte Carlo policy gradient algorithm이다.

다만 policy-based algorithm은 trajectory sampling에 의존하기 때문에 gradient variance가 크고 sample efficiency가 낮아질 수 있다.

Value-based Algorithm

Value-based algorithm은 policy를 직접 학습하기보다 value function을 학습한다. 일반적으로 state value function $V^{\pi}(s)$보다 state-action value function $Q^{\pi}(s,a)$를 학습하는 경우가 많다.

State-action value function은 다음과 같이 정의된다.

\[Q^{\pi}(s,a) = \mathbb{E}_{\tau \sim \pi} \left[ \sum_{t=0}^{T} \gamma^t r_t \mid s_0 = s, a_0 = a \right]\]

Value-based algorithm에서는 state $s$에서 $Q^{\pi}(s,a)$가 가장 큰 action $a$를 선택하는 방식으로 policy를 구성할 수 있다.

\[a^* = \arg\max_{a \in \mathcal{A}} Q^{\pi}(s,a)\]

이 방식은 policy를 직접 출력하지 않아도, value function을 통해 action selection을 수행할 수 있다는 장점이 있다.

대표적인 value-based algorithm에는 SARSA, Q-Learning, DQN이 있다. DQN은 neural network를 이용해 $Q$ function을 근사하는 deep reinforcement learning algorithm이다.

Value-based algorithm은 policy-based algorithm보다 sample efficient한 경우가 많다. 하지만 continuous action space에서는 $\arg\max$ 계산이 어려워질 수 있고, optimal policy로의 안정적인 수렴이 항상 보장되는 것은 아니다.

Model-based Algorithm

Model-based algorithm은 environment의 transition dynamics를 학습하거나 이미 알려진 environment model을 사용하는 방법이다.

Transition model은 현재 state $s_t$와 action $a_t$가 주어졌을 때 next state $s_{t+1}$를 예측한다.

\[s_{t+1} \sim P(s_{t+1} \mid s_t, a_t)\]

Model-based algorithm에서 agent는 learned model을 이용해 여러 possible trajectory를 미리 simulation할 수 있다. 이 과정은 실제 environment와 상호작용하지 않고 future state와 reward를 예측하는 방식으로 볼 수 있다.

예를 들어 agent가 현재 state $s_t$에서 여러 action candidate를 비교한다고 하자. Model-based algorithm은 각 action이 만들어낼 future trajectory를 예측하고, 더 높은 expected return을 갖는 action을 선택할 수 있다.

Model-based algorithm은 data collection 비용이 큰 문제에서 유리할 수 있다. Robotics처럼 실제 environment interaction이 비싸거나 위험한 경우, model을 이용한 planning은 sample efficiency를 높일 수 있다.

하지만 정확한 environment model을 학습하는 것은 어렵다. Model error가 여러 time step에 걸쳐 누적되면 long-horizon prediction이 불안정해질 수 있다.

Model-Free and Model-Based Reinforcement Learning

Reinforcement Learning algorithm은 environment model을 사용하는지에 따라 model-free와 model-based로 나눌 수 있다.

Type Description Example
Model-free RL Transition model $P(s_{t+1} \mid s_t, a_t)$를 직접 학습하거나 사용하지 않고 policy 또는 value function을 학습하는 방법 REINFORCE, A2C, PPO, DQN, SAC
Model-based RL Transition model을 학습하거나 주어진 model을 이용해 future trajectory를 예측하고 planning하는 방법 MPC, MCTS, World Models, AlphaZero

Model-free RL은 environment model 없이 experience로부터 policy 또는 value function을 학습한다. 따라서 구현이 상대적으로 단순하고 다양한 environment에 적용하기 쉽다.

Model-based RL은 transition model을 이용해 future를 예측할 수 있다. 이 때문에 sample efficiency가 높아질 수 있지만, model prediction error를 관리해야 한다.

On-policy and Off-policy

Reinforcement Learning algorithm은 training data를 어떻게 사용하는지에 따라 on-policy와 off-policy로도 구분된다.

On-policy algorithm은 현재 policy $\pi$로부터 생성된 data만 사용하여 현재 policy를 update한다. 즉, data를 수집한 policy와 update되는 policy가 같다.

Off-policy algorithm은 현재 policy와 다른 policy로부터 수집된 data도 update에 사용할 수 있다. 즉, 과거에 수집한 experience를 replay buffer에 저장하고 재사용할 수 있다.

Type Data usage Advantage Limitation Example
On-policy 현재 policy $\pi$로 생성한 data만 사용 학습 objective와 data distribution이 일치함 수집한 data를 재사용하기 어려워 sample inefficient함 REINFORCE, A2C, PPO
Off-policy 과거 policy 또는 다른 behavior policy로 생성한 data도 사용 experience replay가 가능해 sample efficient함 data distribution mismatch를 고려해야 함 Q-Learning, DQN, SAC

On-policy에서는 policy가 update되면 이전 policy로 수집한 data는 현재 policy의 data가 아니게 된다. 따라서 많은 algorithm에서 이전 data를 버리고 새 policy로 다시 trajectory를 수집한다.

Off-policy에서는 이전에 수집한 experience를 여러 번 재사용할 수 있다. 이 때문에 sample efficiency는 좋아질 수 있지만, 현재 policy와 data를 만든 behavior policy 사이의 차이를 고려해야 한다.

Relation to REINFORCE

REINFORCE는 policy-based, model-free, on-policy algorithm으로 분류할 수 있다.

Property REINFORCE
Main target Policy $\pi_{\theta}$
Model usage Model-free
Data usage On-policy
Optimization Policy gradient
Return estimation Monte Carlo return

REINFORCE는 environment model $P(s_{t+1} \mid s_t, a_t)$를 학습하지 않는다. 대신 현재 policy $\pi_{\theta}$로 trajectory $\tau$를 sampling하고, 그 trajectory에서 얻은 return $R(\tau)$를 이용해 policy parameter $\theta$를 update한다.

따라서 REINFORCE를 이해하려면 다음 개념이 필요하다.

  1. Policy $\pi_{\theta}$가 state $s_t$에서 action $a_t$를 sampling한다.

  2. Sampling된 action으로 trajectory $\tau$가 생성된다.

  3. Trajectory에서 return $R(\tau)$를 계산한다.

  4. Expected return $J(\pi_{\theta})$가 커지도록 policy parameter $\theta$를 update한다.

즉, REINFORCE는 value function이나 transition model을 먼저 학습하는 방식이 아니라, sampled trajectory의 return을 이용해 policy를 직접 개선하는 algorithm이다.

Summary

Reinforcement Learning algorithm은 학습 대상에 따라 policy-based, value-based, model-based algorithm으로 나눌 수 있다.

Policy-based algorithm은 policy $\pi$를 직접 학습하며, 대표적인 예시는 REINFORCE이다.

Value-based algorithm은 $Q^{\pi}(s,a)$와 같은 value function을 학습하고, value가 큰 action을 선택하는 방식으로 policy를 구성한다.

Model-based algorithm은 transition model $P(s_{t+1} \mid s_t, a_t)$를 학습하거나 사용하여 future trajectory를 예측한다.

Model-free RL은 environment model 없이 policy 또는 value function을 학습하는 방식이다.

Model-based RL은 environment model을 이용해 planning 또는 simulation을 수행하는 방식이다.

On-policy algorithm은 현재 policy로부터 생성된 data만 사용하여 policy를 update한다.

Off-policy algorithm은 과거 policy 또는 다른 behavior policy로부터 수집한 data도 재사용할 수 있다.

REINFORCE는 policy-based, model-free, on-policy algorithm으로 이해할 수 있다.
